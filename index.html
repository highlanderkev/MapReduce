Custom Map Reduce

There are two main apps which both perform a simple word count on text files. One directory is for a map reduce program that creates actors all on the same machine. The second directory is for a remote map reduce program where some of the actors are on remote machines.

The application has two main functions or actors. The first (map) actor implements the map function and outputs the word and count to the (reduce) actor. The reduce actor then combines words and counts and outputs the final key/values of word and the total count of how many timees the word appears in the text.

Next Step:

The next step or challenge is to then transform either one of these map/reduce applications into an application which implements a higher-order mapreduce implementation, which can take any map and reduce function and run it on the files.
